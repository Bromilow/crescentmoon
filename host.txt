=======================================================
STEP 6 — UPDATE .htaccess TO SERVE STATIC SITE
=======================================================

In FileZilla, find the existing .htaccess file in /public_html/.
Right-click → View/Edit (opens in a text editor).

REPLACE the entire contents with the following:

----------- START OF .htaccess -----------

# Crescent Moon Insulation — Static Site on GoDaddy Managed WP Hosting
# SSL is handled by GoDaddy — no HTTPS redirect needed here

Options -Indexes

# Serve static HTML files (priority over WordPress index.php)
DirectoryIndex index.html index.php

# Serve static pages — map clean URLs to .html files
RewriteEngine On

# Skip rewriting for actual files and directories
RewriteCond %{REQUEST_FILENAME} -f [OR]
RewriteCond %{REQUEST_FILENAME} -d
RewriteRule ^ - [L]

# Serve .html files for extensionless URLs
RewriteCond %{REQUEST_FILENAME}.html -f
RewriteRule ^(.*)$ $1.html [L]

# Custom 404
ErrorDocument 404 /404.html

# Enable Gzip Compression
<IfModule mod_deflate.c>
  AddOutputFilterByType DEFLATE text/html
  AddOutputFilterByType DEFLATE text/css
  AddOutputFilterByType DEFLATE application/javascript
  AddOutputFilterByType DEFLATE image/svg+xml
</IfModule>

# Browser Caching
<IfModule mod_expires.c>
  ExpiresActive On
  ExpiresByType image/webp       "access plus 1 year"
  ExpiresByType image/jpeg       "access plus 1 year"
  ExpiresByType image/png        "access plus 1 year"
  ExpiresByType image/svg+xml    "access plus 1 year"
  ExpiresByType image/x-icon     "access plus 1 year"
  ExpiresByType text/css         "access plus 1 month"
  ExpiresByType application/javascript "access plus 1 month"
  ExpiresByType text/html        "access plus 1 hour"
  ExpiresByType font/woff2       "access plus 1 year"
</IfModule>

# Security Headers
<IfModule mod_headers.c>
  Header set X-Content-Type-Options "nosniff"
  Header set X-Frame-Options "SAMEORIGIN"
  Header set Referrer-Policy "strict-origin-when-cross-origin"
</IfModule>

----------- END OF .htaccess -----------

Save the file and upload it back (FileZilla will ask to upload the changes).

NOTE: If GoDaddy's Managed WordPress plan locks the .htaccess and
prevents modification, see the TROUBLESHOOTING section at the bottom.

=======================================================
TROUBLESHOOTING
=======================================================

PROBLEM: .htaccess is locked / can't be edited
SOLUTION: Contact GoDaddy support and ask them to allow .htaccess
modification on your Managed WordPress plan. Alternatively, ask them
to switch your plan to "Web Hosting" (cPanel-based), which gives
full .htaccess control and is better suited for static sites.

PROBLEM: Homepage still shows WordPress default page
SOLUTION: In FileZilla, rename the existing /public_html/index.php
to /public_html/index.php.bak — this forces the server to use your
index.html instead.

PROBLEM: Pages return 404 (clean URLs not working)
SOLUTION: Ensure mod_rewrite is enabled on your server.
Contact GoDaddy support to confirm mod_rewrite is active on your plan.

PROBLEM: Images not loading
SOLUTION: Check file paths. All image references in HTML should use
relative paths from the file's location, e.g., ../assets/images/photo.webp

PROBLEM: SSL not showing (no padlock)
SOLUTION: Go to GoDaddy Managed WordPress dashboard → Settings →
SSL/TLS → Force HTTPS → Enable.
